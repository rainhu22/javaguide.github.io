Git基础

Git是当前最先进、最主流的分布式版本控制系统，核心能力面向代码文件的版本控制，代码的任何修改历史都会被记录管理起来，意味着可以恢复到到以前的任意时刻状态。支持跨区域多人协作编辑，是团队项目开发的必备基础，所以Git也就成了程序员的必备技能。

git工作流程

  • 准备仓库：git clone创建或从服务端克隆一个仓库。
  • 编写代码：在工作目录中添加、修改代码。
  • 暂存git add：将需要进行版本管理的文件放入暂存区域。
  • 提交git commit 或者 git commit -a 直接add加commit一起完成：将暂存区域的文件提交到本地Git仓库。
  • 获取更新fetch/pull：从服务端更新到本地，获取他人推送的更新，与他人协作、共享
  • 推送git push：将本地仓库推送到远程仓库，同步版本库。

  • 取消暂存的文件（已经add）

  • git fetch、git pull 都是从远程服务端获取最新记录，区别是git pull多了一个步骤，就是自动合并更新工作区。

  • git checkout . 、git checkout [file] 会清除工作区中未添加到暂存区的修改，用暂存区内容替换工作区。

  • git diff用来比较不同文件版本之间的差异。

  • git branch dev创建了一个名字为dev的分支

  • git switch feature git checkout [-b(创建新的分支并切换)] feature用于切换分支

  • git merge用于合并分支

  • 每一个提交都有一个唯一标识，主要就是提交的hash值commit id，在很多指令中会用到，如版本回退、拣选提交等，需要指定一个提交。那标识唯一提交有两种方式：

    • 首先就是commit id，一个40位编码，指令中使用的时候可以只输入前几位（6位）即可。

    • 还有一种就是HEAD~n，是基于当前

      位置的一个相对坐标。

      • HEAD 表示当前分支的最新版本，是比较常用的参数。
      • HEAD^上一个版本，HEAD^^ 上上一个版本。
      • HEAD~ 或HEAD~1 表示上一个版本，以此类推，HEAD^10 为最近第10个版本。
      • HEAD@{2}在git reflog日志中标记的提交记录索引。

  • git log、git reflog可以查看历史日志，可以看每次提交的唯一编号（hash）。区别是git reflog可以查看所有操作的记录（实际是HEAD变更记录），包括被撤销回退的提交记录。
